# 前端开发

本框架采用了模块化的前端开发模式，旨在提高代码的可维护性和可扩展性。

## 1. 模块结构

如果您正在开发一个新的业务模块（例如 `products`），请遵循以下目录结构：

```text
src/pages/products/
├── components/          # 模块私有组件
├── Products.tsx         # 主页面
├── ProductDetail.tsx    # 详情页
├── routes.tsx           # 路由配置文件
└── menu.ts              # 菜单配置文件
```

## 2. 创建页面组件

在模块目录下创建页面文件，文件命名遵循**大驼峰**。

```tsx
// src/pages/products/Products.tsx
import { Card, CardContent, CardHeader, CardTitle } from '@/components/ui/card';

export function Products() {
  return (
    <div className="space-y-6">
      <h1 className="text-3xl font-bold">产品管理</h1>
      <Card>
        <CardHeader>
          <CardTitle>产品列表</CardTitle>
        </CardHeader>
        <CardContent>
          {/* 功能实现代码 */}
        </CardContent>
      </Card>
    </div>
  );
}
```

## 3. 配置模块路由

在模块目录下创建 `routes.tsx`，导出该模块的所有路由配置。

```tsx
// src/pages/products/routes.tsx
import { Route } from 'react-router-dom';
import { Products } from './Products';

export const ProductRoutes = (
  <>
    <Route path="products/list" element={<Products />} />
    {/* 其他路由... */}
  </>
);
```

然后，在 `src/App.tsx` 中注册引用：

```tsx
// src/App.tsx
import { ProductRoutes } from '@/pages/products/routes';

// ... 在 Routes 中引用
<Route element={<ProtectedRoute />}>
  <Route path="/" element={<MainLayout />}>
    <Route index element={<Dashboard />} />
    {ProductRoutes} {/* 注册模块路由 */}
  </Route>
</Route>
```

## 4. 配置模块菜单

在模块目录下创建 `menu.ts`，导出菜单配置。

```typescript
// src/pages/products/menu.ts
import { PackageIcon } from '@heroicons/react/24/outline';

export const productMenu = {
  title: '产品管理',
  icon: PackageIcon,
  userOnly: true,
  children: [
    { title: '产品列表', path: '/products/list' },
  ],
};
```

然后，在 `src/components/layout/Sidebar.tsx` 中注册引用：

```tsx
// src/components/layout/Sidebar.tsx
import { productMenu } from '@/pages/products/menu';

const menuItems: MenuItem[] = [
  // ... 其他菜单
  productMenu, // 注册模块菜单
];
```

## 5. 开发规则总结

- **页面命名**：遵循大驼峰（如 `Products.tsx`）。
- **组件位置**：模块私有组件存放在模块目录下的 `components/`。
- **路径规范**：页面的访问路径通常为 `/{module}/{page}`。
- **解耦设计**：尽量保持模块配置（路由和菜单）的独立性，通过导入的方式集成到主框架中。
